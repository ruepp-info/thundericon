    Unread messages (Cards view)

    Make unread messages stand out more than Thunderbird's default bold text — much easier to scan in dark mode. Applies to the Cards layout only; the Table layout is left unchanged.

    Accent color

    The accent color is used for the bar and the ring; the Fade read style doesn't use it.

    Domain colors

    Force a specific color for senders from a given domain (overrides the mode above).

    Brand logos (BIMI)

    When a sender's domain publishes a BIMI logo and the message passes DMARC authentication, show the brand logo in place of the initials; everyone else keeps initials. Looking up logos contacts your DNS resolver and the brand's server, so this is off by default.

    Resolve the logo on the sender's registrable domain instead of the exact sending domain — e.g. a message from test@trx.mail2.disneyplus.com looks up disneyplus.com. Helps brands that send from subdomains but publish BIMI only on their main domain.

    Custom DoH endpoint URL Works with any standard RFC 8484 DoH resolver (e.g. AdGuard, Quad9, NextDNS, Cloudflare, or a self-hosted endpoint).

    Thunderbird's system resolver can't read the TXT records BIMI needs, so lookups fall back to DNS-over-HTTPS via the chosen provider. The queried sender domain is sent to that provider.

    Don't look up logos in these folders:
    Cached logos

    Clearing forces every logo to be re-fetched on next view.

    Troubleshooting

    Opens a tool to check whether a given domain publishes a BIMI logo, with a step-by-step log.

    Profile photos (Gravatar)

    When a sender has a Gravatar profile photo for their address, show it in place of the initials; everyone else keeps initials. Profile photos take precedence over BIMI brand logos. A lookup sends a hash of the sender's email address to gravatar.com, so this is off by default.

    People change their Gravatar rarely, so a long refresh keeps lookups to a minimum. Negative results (no photo) are cached for the same period.

    Don't look up photos in these folders:
    Cached photos

    Clearing forces every photo to be re-fetched on next view.
